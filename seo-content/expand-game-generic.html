Script Troubleshooting Guide — Common Issues & Solutions

Script Not Loading After Execution

If a script does not appear to load after clicking Execute in Delta Executor, try these solutions: wait 10-15 seconds as some scripts take time to initialize; check that injection was successful before executing; try re-injecting Delta and executing again; ensure you are using the latest Delta version; verify the script is compatible with the current game version.

GUI Not Appearing or Appearing Off-Screen

Script GUIs sometimes spawn outside the visible screen area, especially on different display resolutions. Most scripts have a key bind to toggle the GUI — typically the Home key or right Shift key on PC, or a floating button on mobile. Try pressing these keys to show or reposition the GUI.

Features Not Working Despite GUI Loading

If the script GUI loads but individual features do not seem to work, the script may need updating for the latest game version. Check RoCrack for an updated version of the script. Also ensure you have properly waited for the game to fully load before injecting and executing.

Game Crashing After Script Execution

Game crashes typically occur when scripts conflict with specific game mechanics or when too many features are enabled simultaneously on resource-limited devices. Try disabling resource-intensive features, enable features one at a time to identify the problematic one, and ensure your device meets minimum requirements for the script.

Script Working Inconsistently

If a script works sometimes but not others, the issue is often related to timing — inject and execute after the game is completely loaded, not during loading screens or transitions. Server-specific issues can also cause inconsistency; try switching to a different game server.
